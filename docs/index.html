Dance Project Classification

This is a classification algorithm that can classify keypoint data of body movements in dance videos into 2 different dance styles. The two dance styles it will classify are Jazz Ballet and House.

Python scikit-learn numpy pandas matplotlib.pyplot seaborn pickle jupyter
More details

Problem

This project builds a classification algorithm that can classify keypoint data of body movements in dance videos into two different dance styles: Jazz Ballet and House.

Approach

This project will build a classification algorithm that can classify keypoint data of body movements in dance videos into 2 different dance styles. Specifically, I will use keypoint data which identifies major joints and key facial features and tracks their locations throughout a video. Then, I will build a feature vector using various calculations on the keypoint data. These calculations will be determined in order to identify a dance such as velocity or "energy," range of motion, etc. I can then use the feature vectors to build a machine learning model to classify dance videos into two different styles. I start with a small number of features and add more as needed to improve classification. This is a supervised learning task.

Results & Impact

After this analysis, I have determined that the best model for this task is logistic regression after scaling with StandardScaler and using PCA for dimension reduction. The accuracy level of the best model was 0.91 and the f1-scores for ballet-jazz and house are 0.90 and 0.91, respectively. These numbers met my goals of 0.90 for the accuracy level and f1-scores. The close second model is an SVM because it achieved the same scores, but just had a longer computing time.

Concrete Cracks Classification Project

This data used for this project is a set of images of cracked and non-cracked concrete from walls, sidewalks, and bridges. The project builds a classification algorithm to classify images into cracked or non-cracked.

numpy pandas matplotlib seaborn scipy pillow scikit-learn
More details

Problem

This project uses the Concrete Cracks dataset. The goal is to classify images into cracked or non-cracked concrete to be able to quickly identify locations with concrete cracks.

Approach

To address our research questions, we began by preprocessing the data. This involved combining the images from the two datasets into a single unified dataset. We then converted the categorical labels into numerical values and flattened the image data to prepare it for model input. After preprocessing, we analyzed the shape and structure of the dataset to determine which classification models would be most appropriate. We proceeded to build and evaluate several models, ultimately focusing on logistic regression and a random forest classifier to compare performance and identify the most effective approach for our specific dataset.

Results & Impact

In this project, we explored the effectiveness of various classification models in identifying cracks in images. Through preprocessing, model experimentation, and hyperparameter tuning, we observed that class imbalance significantly impacted our model performance, particularly in overpredicting the majority class. By implementing balancing techniques and using tools like GridSearchCV, we were able to develop more reliable models that better captured patterns across both cracked and non-cracked images. In the end the model that performed the best was the Random Forest Classifier due to its higher performance across all metrics.
